# Web Server (Веб-сервер)

Минимальный HTTP-сервер на Python, реализованный с использованием `socket` в учебных целях. Поддерживает базовый роутинг (аналогично FastAPI) и обработку текстовых HTTP-запросов.

# Функциональность
- Регистрация маршрутов (routing)
- Обработка HTTP GET-запросов через интерфейс: `app.get("/items")`
- Механизм обработки HTTP-исключений с возвратом соответствующего `status_code` и сообщения об ошибке (`detail`)

# Установка и запуск
Требования: `Python`  
  
Установка зависимостей из `pyproject.toml` зависит от используемого менеджера пакетов:  

**Использование "Poetry"**  
 ``` poetry install```  
**Использование "pip"**  
 ``` pip install .```  
**Использование "uv"**  
 ``` uv sync ```\
 ```uv pip install```  
  
Запуск проекта происходит через `main.py`

# Использование
Сервер запускается через выполнение скрипта `main.py` 
На примере менеджера пакетов `uv`:
`uv run main.py`

Пример регистрации маршрутов (`main.py`)
```python
@app.get(path="/items/{item_id}")  
def read_item(item_id: int, q: Union[str, None] = None) -> dict[str, int | str | None]: 
    return {"item_id": item_id, "q": q}
```
- создание маршрута с параметром пути (`item_id`) и query-параметром (`q`).
```python
@app.get("/div")  
def div(a: float, b: float) -> dict[str, float]:  
    if b == 0:  
        raise HTTPException(status_code=400, detail="Деление на ноль невозможно")  
    return {"result": a / b}
```
- создание маршрута для выполнения деления с query-параметрами (`a`, `b`) и обработкой ошибки деления на ноль
